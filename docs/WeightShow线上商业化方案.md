# WeightShow 线上增长与商业化方案

> 版本：2026-05-31  
> 范围：仅考虑线上方向；线下合作暂不考虑。  
> 核心思路：先把小程序做成更好用、更美观、更实用的减脂工具，再用减肥群、打卡活动和机器人自动运营做付费转化。

## 1. 方向校准

当前最适合 WeightShow 的路线不是“猛加广告”，也不是“重人工运营微信群”，而是：

**工具体验升级 → 用户增长与留存提升 → 付费减肥群/活动 → 机器人自动总结、激励、排名 → 形成差异化收费**

理由：

- 体重打卡是高频工具，核心体验不能被广告破坏。
- 微信群运营很耗人力，不能依赖人工每天发内容。
- 现有用户已经有稳定打卡行为，适合做“自动化挑战”和“机器人监督”。
- 小程序已有 PK、排行榜、连续打卡、机器人、报告、分享海报等基础能力，适合继续打磨成完整工具。

## 3. 总体策略

### 阶段一：把小程序做好

目标：让用户觉得这个工具值得长期放在微信里使用。

重点：

- 更美观
- 更清晰
- 更实用
- 更有目标感
- 更有成就感
- 更容易分享

### 阶段二：把打卡变成活动

目标：让用户从“自己记录”进入“跟一群人一起坚持”。

重点：

- 自动参与月度挑战
- 自动统计排名
- 自动生成战报
- 自动展示连续打卡和减重成果

### 阶段三：把活动变成付费群

目标：用机器人能力区分免费群和付费群。

免费群：

- 基础交流
- 基础打卡提醒
- 偶尔活动通知

付费群：

- 每日机器人总结
- 每日群排名
- 每日激励
- 每周战报
- 连续打卡奖励
- PK 队伍数据
- 机器人自动点名未打卡用户

## 4. 产品升级方向

### 4.0 近期必须补齐的小功能

这些功能优先级很高，因为它们直接决定小程序是不是“好用的工具”，也会影响后续活动和付费群的转化。

| 功能 | 当前状态 | 建议 |
|---|---|---|
| 查看个人所有打卡记录 | `records` 页面存在，但页面基本是占位 | 做成正式的历史记录页，支持按月查看、搜索、编辑、图片预览 |
| 减肥报告 | 已有年度报告基础 | 扩展为月度报告、区间报告，支持从个人主页/记录页进入 |
| 分享朋友圈 | 首页和年报已有分享逻辑 | 做成更明确的分享入口，报告和战绩海报优先支持朋友圈 |
| 上传图片/拍照记录 | 首页打卡弹窗里已有，但入口藏在输入框小图标里 | 暂不作为近期重点，保留现有能力即可 |

推荐先做历史记录、减肥报告和朋友圈分享，不急着做复杂付费系统。它们会让小程序的工具价值明显提升。

### 4.1 首页升级

首页是核心打卡页面，应优先提升体验。

建议增加：

- 今日打卡状态更明显。
- 当前体重、目标体重、距离目标还差多少。
- 连续打卡天数。
- 本月已打卡天数。
- 本月体重变化。
- 一个更好看的体重趋势图。
- 轻量化的“本月挑战进度”模块。

首页不要做太多广告，也不要频繁弹窗。

推荐首页结构：

1. 今日状态：已打卡/未打卡。
2. 当前体重与目标差距。
3. 本月趋势图。
4. 连续打卡和本月打卡进度。
5. 月度挑战入口。

### 4.1.1 历史打卡记录页

当前 `pages/records` 页面存在，但体验还没有完成。建议把它做成正式的“记录”页。

基础能力：

- 查看全部打卡记录。
- 按月份筛选。
- 显示日期、体重、体重变化、BMI。
- 支持编辑体重。
- 支持删除记录。

进阶能力：

- 本月记录汇总。
- 连续打卡标识。
- 体重上升/下降颜色提示。
- 点击某条记录进入详情。

这个页面对用户很重要，因为它让用户觉得数据是“可管理、可回顾”的，而不是只在首页日历里点来点去。

### 4.2 月度报告

现在已有年度报告，建议升级为“月度报告”。

月度报告比年度报告更适合高频使用和分享。

报告内容：

- 本月打卡天数。
- 最长连续打卡。
- 本月体重变化。
- 最高/最低体重。
- 距离目标体重还差多少。
- BMI 变化。
- 本月自律标签。
- 一张可分享海报。

月度报告的价值：

- 提升用户成就感。
- 增加分享传播。
- 为付费群活动提供战报素材。
- 后续可做高级报告或模板付费。

### 4.2.1 报告入口

报告不要只放在“我的”页或年度活动入口里，建议增加几个稳定入口：

- 首页：本月报告卡片。
- 历史记录页：查看本月报告。
- 排行榜页：查看我的战绩。
- 我的页：我的减肥报告。

报告类型建议：

| 报告 | 用途 |
|---|---|
| 月度报告 | 高频回顾和分享 |
| 年度报告 | 年终传播活动 |
| 区间报告 | 活动/付费群结营使用 |
| PK 报告 | 群和活动战报 |

### 4.2.2 朋友圈分享

分享功能要围绕报告和成果，而不是只分享小程序首页。

优先支持：

- 月度报告分享朋友圈。
- 本月已瘦 X 斤分享朋友圈。
- 连续打卡 X 天分享朋友圈。
- 7 天/21 天挑战结营海报。
- PK 队伍战报海报。

分享文案建议：

- `本月已打卡 X 天，继续记录变化`
- `我已经连续打卡 X 天，一起坚持`
- `本月体重变化 X 斤，来和我一起打卡`
- `我的 X 月减肥报告生成了`

注意：朋友圈分享不要强弹，应该在用户看到成果后自然出现。

### 4.3 自动挑战

建议做“小程序内自动挑战”，不需要用户复杂报名。

例如：

**本月自律挑战**

规则：

- 用户只要打卡，即自动参与。
- 连续打卡 3 天获得“小坚持”。
- 连续打卡 7 天获得“自律入门”。
- 连续打卡 14 天获得“稳定输出”。
- 连续打卡 21 天获得“月度达人”。
- 连续打卡 30 天获得“满勤王”。

这类机制不需要人工运营，但能增强留存。

### 4.4 排行榜升级

当前排行榜可以继续强化。

建议拆成：

- 总打卡榜
- 本月打卡榜
- 连续打卡榜
- 本月减重榜
- PK 队伍榜

更适合活动和付费群的榜单：

- 今日已打卡名单
- 今日未打卡名单
- 本周坚持榜
- 本月挑战榜

### 4.5 PK 功能升级

PK 是社群付费活动的基础。

建议增强：

- 队伍目标。
- 队伍完成率。
- 成员今日打卡状态。
- 队伍周报。
- 队伍月报。
- 队伍机器人绑定状态。

付费群可以围绕 PK 组织：

- 一个付费群就是一个挑战队。
- 机器人每天自动发队伍战报。
- 小程序展示队伍成员数据。

### 4.6 机器人能力升级

机器人是付费活动的核心差异化。

免费群机器人：

- 每日基础提醒。
- 简单关键词回复。

付费群机器人：

- 每日早上激励。
- 晚上未打卡提醒。
- 每日打卡总结。
- 群成员排名。
- 本周战报。
- 本月战报。
- 点名连续打卡用户。
- 点名未打卡用户。
- 根据体重变化生成一句反馈。

机器人要尽量自动化，减少人工参与。

## 5. 付费群与活动设计

### 5.1 免费群定位

免费群不用强运营，主要做承接：

- 用户交流。
- 活动通知。
- 小程序功能反馈。
- 引导进入付费挑战。

免费群不要承诺强监督，否则会耗费人力。

### 5.2 付费群定位

付费群卖的不是“聊天”，而是：

**机器人监督 + 打卡氛围 + 排名反馈 + 阶段性目标**

用户付费的理由：

- 有人/机器人提醒我。
- 我能看到自己排名。
- 我能看到别人也在坚持。
- 每天有反馈。
- 每周有总结。
- 比免费群更有约束感。

### 5.3 第一款活动：7 天体验挑战

建议先做低门槛验证。

| 项目 | 设计 |
|---|---|
| 名称 | 7 天减脂打卡体验营 |
| 价格 | 9.9 元 |
| 周期 | 7 天 |
| 形式 | 微信群 + 小程序打卡 + 机器人总结 |
| 人数 | 每期 30-100 人 |
| 核心任务 | 每天记录体重 |
| 机器人内容 | 早激励、晚提醒、每日排名、最终战报 |

为什么先做 9.9 元：

- 门槛低。
- 用户决策成本小。
- 适合验证是否有人愿意为监督付费。
- 不需要承诺复杂服务。

### 5.4 第二款活动：21 天正式挑战

等 7 天活动跑通后再做。

| 项目 | 设计 |
|---|---|
| 名称 | 21 天自律减脂挑战营 |
| 价格 | 39-69 元 |
| 周期 | 21 天 |
| 形式 | 付费群 + 小程序挑战榜 + 机器人战报 |
| 核心卖点 | 自动监督、排行榜、周报、结营报告 |
| 奖励 | 满勤徽章、优秀用户展示 |

### 5.5 免费群与付费群区别

| 能力 | 免费群 | 付费群 |
|---|---|---|
| 基础交流 | 有 | 有 |
| 活动通知 | 有 | 有 |
| 每日机器人激励 | 少量 | 有 |
| 每日排名 | 无 | 有 |
| 未打卡提醒 | 无 | 有 |
| 每周战报 | 无 | 有 |
| 结营报告 | 无 | 有 |
| PK 队伍数据 | 无 | 有 |
| 机器人个性化反馈 | 无 | 有 |

## 7. 广告策略

广告只做轻量，不破坏核心打卡体验。

不建议：

- 首页强插屏。
- 打卡按钮前弹广告。
- 用户刚打开就弹广告。
- 每次打卡成功都强制广告。

可以考虑：

- 排行榜底部广告。
- 报告页底部广告。
- 我的页底部广告。
- 海报生成完成后的轻广告。
- 免费用户看高级报告前的激励广告。

广告目标：

- 当前约 12 元/天。
- 短期目标 15-20 元/天。
- 不为广告牺牲留存。

## 10. 30 天执行计划

### 第 1 周：产品体验梳理

目标：先把工具感做强。

任务：

- 梳理首页信息层级。
- 设计新版首页。
- 设计历史打卡记录页。
- 明确月度报告字段。
- 明确本月挑战规则。
- 梳理排行榜升级方案。

产出：

- 首页改版方案。
- 历史记录页方案。
- 月度报告方案。
- 自动挑战规则。

### 第 2 周：首页和历史记录

任务：

- 首页展示当前体重、目标体重、连续打卡、本月进度。
- 上线历史打卡记录页。

目标：

- 让基础记录体验明显变好。
- 降低用户找功能的成本。

### 第 3 周：月度报告和朋友圈分享

任务：

- 上线月度报告基础版。
- 增加报告海报分享。
- 优化朋友圈分享文案和入口。

目标：

- 提高用户打开后的满意度。
- 增加分享素材。

### 第 4 周：机器人战报和免费挑战

任务：

- 机器人支持每日群总结。
- 机器人支持群排名。
- 免费群测试 7 天挑战。
- 观察用户是否愿意跟着机器人节奏打卡。

目标：

- 跑通“群 + 小程序 + 机器人”的自动化流程。

### 第 5 周：付费体验营

任务：

- 上线 9.9 元 7 天体验营。
- 区分免费群和付费群权益。
- 机器人每天自动发付费群战报。
- 结营生成报告。

目标：

- 验证是否有人愿意为机器人监督和群氛围付费。

## 11. 当前最推荐的第一步

第一步不是加广告，也不是马上卖课，而是：

**新版首页 + 历史打卡记录页 + 月度报告 + 朋友圈分享**

这三个功能会直接提升工具价值，也能为后续付费群活动提供基础：

- 首页让用户更愿意长期用。
- 历史记录页让用户能完整管理自己的数据。
- 月度报告让用户更有成就感和分享动力。
- 朋友圈分享让增长更自然。

然后再做：

**本月自动挑战 + 7 天付费体验营 + 机器人每日总结/激励/排名**

这样商业化是建立在产品价值上的，而不是硬推广告或硬拉群。
