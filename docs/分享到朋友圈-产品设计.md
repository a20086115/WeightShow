# 首页与分享相关产品定义

> 仅产品设计，不含开发细节。确认后再进入开发。  
> 涵盖：首页样式与入口、分享/海报、新用户引导、海报视觉优化。

---

## 1. 产品目标

- **做什么**：在首页支持用户把小程序「分享到朋友圈」，让好友在朋友圈里看到一条条小程序卡片，形成列表式曝光，带来自然拉新。
- **不做成**：不改变现有「转发给好友」逻辑，只是新增「分享到朋友圈」这一条出口。

---

## 2. 用户故事

| 角色 | 场景 | 期望 |
|------|------|------|
| 正在减肥/打卡的用户 | 本月瘦了一点，想晒一下 | 能一键分享到朋友圈，文案里带上「本月瘦了 X 斤」之类，有成就感、也吸引别人点进来 |
| 刚打完卡的用户 | 今天第一次打卡，想留个记录 | 可选：打卡成功后看到轻提示「分享到朋友圈，记录这一天」 |
| 好友 / 路人 | 刷朋友圈看到一条「本月已瘦 X 斤，一起打卡！」 | 点击卡片进入小程序首页，有机会成为新用户 |

### 3.1 基础能力：支持从首页分享到朋友圈

- **行为**：用户在首页时，点击右上角「…」，菜单里出现「分享到朋友圈」；点击后，会发出一条朋友圈，展示为小程序卡片（标题 + 配图），好友点击进入小程序首页。
- **列表效果**：每条分享的标题可以不一样（见下文案规则），所以朋友圈里会呈现「多条不同标题的小程序卡片」，即你说的列表效果。

### 3.3 分享卡片配图

- **方案 A**：不自定义配图，使用微信默认的小程序卡片图。  
  优点：无开发量；缺点：每条朋友圈长得一样，列表感弱一点。
- **方案 B**：使用自定义配图（例如本月体重曲线 + 一句「本月瘦了 X 斤」+ 昵称）。  
  优点：列表感强、点击率可能更高；缺点：需要「生成海报图」并在分享时用这张图，逻辑和开发量更大。

**请确认：本期选哪种？** □ A（默认图）  □ B（自定义海报图）  □ 本期不做决定，先做 A，后续再迭代 B

- **点击「去分享」**：仅做引导（例如提示「点击右上角 …，选择分享到朋友圈」），不强制调起分享。

## 4. 首页样式与入口

### 4.1 分享按钮放哪里？与「生成海报」的关系

**现状**：BMI 指数右侧有一个「分享」图标（弯曲箭头），点击后走的是**生成海报**逻辑（弹窗预览海报 → 保存本地），容易让人误以为是「分享到好友/朋友圈」，且和「分享到朋友圈」能力混在一起不清晰。

**可选方案**（请选一种或组合）：

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **A** | 保留现位置，但改图标/文案：图标改为「海报」或「生成长图」，点击 = 生成海报；分享到朋友圈仅通过右上角「…」 | 入口不变，语义清晰 | 首页没有显性的「分享到朋友圈」按钮 |
| **B** | 拆成两个入口：①「生成本月海报」放在图表下方或 BMI 行最右；②「分享」仅指右上角 … → 分享给好友/朋友圈 | 海报与分享分离，各司其职 | 多一个入口，需占一点空间 |
| **C** | 一个入口「分享/海报」：点击后先弹底部 ActionSheet「保存海报」「分享到朋友圈」「分享给好友」，再分别执行 | 一个入口搞定多种分享 | 多一步选择，逻辑稍复杂 |

**关于「分享到朋友圈」是否在首页有按钮**：微信规范下，分享到朋友圈主要依赖右上角「…」；若在首页增加「分享到朋友圈」按钮，点击后只能引导用户点右上角，或先生成海报再引导保存后去相册发朋友圈（无法直接调起发朋友圈）。

## 5. 新用户引导

**现状**：首页未区分新老用户，没有在首次进入时集中收集「身高、目标体重、当前体重」，导致 BMI 无法算、目标体重线缺失，且用户要自己发现「点 BMI」或「点日历某天」才能录入。

**建议**：新用户（例如从未填过身高，或从未有过体重记录）首次进入首页时，弹出**一次性的引导弹框**，顺序收集：

1. **身高**（必填，单位 cm）  
2. **目标体重**（选填，单位斤或 kg 需统一）  
3. **当前体重**（选填，可与「今天」的打卡合并：填完即视为今天已打卡一条）

弹框样式：单页多字段或分步（第一步身高 → 第二步目标体重 → 第三步当前体重），最后「完成」关闭，并可选「不再提示」（仅针对已填过身高的用户再次进入时不再强弹）。

- 弹框内必填项：□ 仅身高  □ 身高 + 目标体重  □ 身高 + 目标体重 + 当前体重

## 6. 生成海报样式优化

**现状（结合你提供的海报截图）**：  
- 顶部黄色波点区域 + 一句成就文案「您的好友【xxx】在【2026-01】期间体重减少4.45KG!」  
- 中间体重/BMI 折线图 + 图例  
- 底部二维码 +「长按识别图中二维码 来和我一起打卡呗~」  
- 底部按钮「取消」「保存本地」  

**可优化方向**（产品层面，不涉及具体像素）：

| 维度 | 现状问题 | 优化建议 |
|------|----------|----------|
| **整体气质** | 黄色波点偏「活动/促销」，和瘦身打卡的简洁、健康感略违和 | 改为白底 + 主色（如品牌蓝）点缀，或浅灰/浅蓝背景，更干净 |
| **成就文案** | 字体偏小、信息密度高，「您的好友」在 C 位略弱化个人成就感 | 主文案突出「本月瘦了 X KG」或「X 斤」，昵称可弱化或放副标题；字号层级区分主/副 |
| **图表** | 与首页图表一致，信息清晰但占比较大 | 可略缩小图表占比，给成就文案和品牌/二维码留出呼吸感；坐标轴字号保证可读 |
| **二维码区** | 二维码 + 一句说明，略显平 | 可加一句 Slogan（如「一起打卡，记录变化」）+ 小程序名称；二维码尺寸保证长按可识别的下限 |
| **按钮** | 仅「取消」「保存本地」 | 若后续支持「分享到朋友圈」用这张图，可增加「分享到朋友圈」按钮（点击后引导保存再发，或仅引导右上角分享） |

## 7. 边界与约束（产品层面）

- **入口**：以微信规范为准，分享到朋友圈的入口仅为「右上角 … → 分享到朋友圈」，不承诺在首页增加单独的「分享到朋友圈」按钮（除非你明确要求）。
- **从朋友圈打开**：好友从朋友圈点击卡片进入的是小程序首页，且为单页模式（仅当前页）；产品上无需单独区分「从朋友圈来」的首页样式，除非你后续要做统计或运营位。
- **文案与合规**：分享文案需符合平台规范，避免绝对化用语（如「最」等），「本月已瘦 X 斤」以真实数据为准。
