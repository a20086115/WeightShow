# 微信机器人架构设计

## 一、系统架构概览

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    微信机器人平台                        │
│              (一个机器人账号，一个skw应用)               │
└─────────────────────────────────────────────────────────┘
                          │
                          │ HTTP POST (webhook)
                          ▼
┌─────────────────────────────────────────────────────────┐
│              云函数 wechatBotNew                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │ 绑定处理模块  │  │ 打卡触发模块 │  │ 定时推送模块 │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
│  ┌──────────────┐  ┌──────────────┐                    │
│  │ 关键词触发   │  │ 话术生成模块 │                    │
│  └──────────────┘  └──────────────┘                    │
└─────────────────────────────────────────────────────────┘
                          │
                          │ 查询/更新
                          ▼
┌─────────────────────────────────────────────────────────┐
│                   云数据库                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │ robotMessages│  │ robotConfigs │  │    users     │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
│  ┌──────────────┐  ┌──────────────┐                    │
│  │      pk      │  │    records   │                    │
│  └──────────────┘  └──────────────┘                    │
└─────────────────────────────────────────────────────────┘
                          │
                          │ 调用
                          ▼
┌─────────────────────────────────────────────────────────┐
│                   小程序端                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │ 绑定界面     │  │ 话术切换界面 │  │ 管理界面     │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 1.2 数据流向

```
用户操作流程：
1. 小程序 → 云函数 → 数据库（绑定、配置）
2. 微信 → 机器人平台 → 云函数 → 数据库（消息处理）
3. 数据库 → 云函数 → 机器人平台 → 微信（消息推送）
```

## 三、核心模块详细设计

### 3.1 绑定模块架构

#### **3.1.1 个人绑定流程**

```
小程序端（pages/robot/index）
    │
    ├─ 显示绑定引导
    ├─ 生成绑定消息：绑定：{用户ID}
    └─ 一键复制功能
         │
         ▼
用户在微信中发送绑定消息
         │
         ▼
机器人平台接收消息
         │
         ▼
云函数 wechatBotNew
    │
    ├─ 解析消息：识别"绑定：用户ID"
    ├─ 查询用户信息（通过用户ID）
    └─ 更新 users 表（mid字段）和 robotConfigs 集合（配置信息）
         │
         ▼
绑定完成，小程序刷新状态
```

#### **3.1.2 PK群绑定流程**

```
小程序端（pages/pk/pk）
    │
    ├─ 显示"绑定机器人"按钮
    ├─ 弹出绑定弹窗
    ├─ 生成绑定消息：绑定：{PK ID}
    └─ 一键复制功能
         │
         ▼
用户在群内发送绑定消息
         │
         ▼
机器人平台接收消息（包含gid和mid）
         │
         ▼
云函数 wechatBotNew
    │
    ├─ 解析消息：识别"绑定：PK ID"
    ├─ 查询PK赛信息（通过PK ID）
    └─ 更新 pk 表（robotBinding字段，包含gid、wechatMid）和 robotConfigs 集合（配置信息）
         │
         ▼
绑定完成，小程序刷新状态
```

### 3.2 打卡触发模块架构

#### **3.2.1 个人打卡触发流程**

```
用户在小程序打卡
    │
    ├─ 插入 records 记录
    └─ 更新 users 表（连续打卡天数等）
         │
         ▼
机器人平台定时请求（每2分钟）
         │
         ▼
云函数 wechatBotNew（定时触发）
    │
    ├─ 查询用户绑定关系（users.mid）和配置信息（robotConfigs）
    ├─ 检查是否开启"个人打卡提醒"
    ├─ 查询用户今日打卡记录
    ├─ 对比上次检查记录（去重）
    ├─ 如果发现新打卡：
    │   ├─ 获取用户数据（体重、BMI等）
    │   ├─ 根据话术类型生成话术
    │   └─ 生成个性化消息
    └─ 返回消息（包含wxid）
         │
         ▼
机器人平台发送消息给好友
         │
         ▼
保存消息记录到 robotMessages
```

#### **3.2.2 PK打卡触发流程**

```
PK成员在小程序打卡
    │
    ├─ 插入 records 记录
    └─ 更新 users 表
         │
         ▼
机器人平台定时请求（每2分钟）
         │
         ▼
云函数 wechatBotNew（定时触发）
    │
    ├─ 查询群聊绑定关系（pk.robotBinding）和配置信息（robotConfigs）
    ├─ 检查是否开启"PK打卡提醒"
    ├─ 查询关联的PK赛
    ├─ 查询PK成员今日打卡记录
    ├─ 找出新打卡的成员（对比上次检查）
    ├─ 如果发现新打卡：
    │   ├─ 获取打卡用户信息、打卡数据
    │   ├─ 根据话术类型生成话术
    │   └─ 生成个性化消息（可@用户）
    └─ 返回消息
         │
         ▼
机器人平台发送消息到群内
         │
         ▼
保存消息记录到 robotMessages
```

### 3.3 定时推送模块架构

```
机器人平台定时请求（每2分钟）
    │
    ├─ [群]主动消息接口（群聊）
    └─ [友/群]主动消息接口（好友）
         │
         ▼
云函数 wechatBotNew
    │
    ├─ 判断当前时间
    │   ├─ 8:00-9:00 → 早上口号推送
    │   ├─ 11:00-12:00 → 中午技巧推送
    │   ├─ 20:00-21:00 → 晚上提醒推送
    │   └─ 21:00后 → 每日战报推送
    │
    ├─ 查询绑定关系
    ├─ 检查是否开启对应时段推送
    ├─ 查询数据：
    │   ├─ 群聊：PK数据、打卡记录
    │   └─ 好友：用户个人数据
    ├─ 根据话术类型生成消息
    └─ 返回消息
         │
         ├─ 群聊：返回消息
         └─ 好友：返回消息+wxid
              │
              ▼
机器人平台发送消息
```

### 3.4 关键词触发模块架构

```
用户在微信发送关键词
    │
    ├─ 群聊：发送到群内
    └─ 好友：发送给机器人
         │
         ▼
机器人平台接收消息
         │
         ▼
云函数 wechatBotNew（关键词触发）
    │
    ├─ 解析关键词
    ├─ 匹配关键词类型：
    │   ├─ "统计" → 统计查询
    │   ├─ "日报" → 日报查询
    │   ├─ "口号" → 口号查询
    │   ├─ "技巧" → 技巧查询
    │   └─ 自定义关键词 → 自定义回复
    │
    ├─ 查询相关数据
    ├─ 根据话术类型生成回复
    └─ 返回消息
         │
         ▼
机器人平台发送回复
```

### 3.5 话术类型模块架构

```
用户在小程序选择话术类型
    │
    ├─ 进入话术类型选择界面
    ├─ 选择话术类型
    └─ 保存到 robotConfigs.speechStyle
         │
         ▼
云函数生成消息时
    │
    ├─ 读取 speechStyle
    ├─ 从话术模板库选择对应模板
    ├─ 替换变量（{weight}、{bmi}等）
    └─ 生成最终消息
```

## 四、数据库架构

### 4.1 核心集合

#### **users 集合 - 添加 mid 字段**（个人绑定）
```javascript
{
  // ... 原有字段 ...
  mid: 'wxid_xxx'                 // 微信用户ID（mid/wxuin），绑定后存储，解绑后清空
}
```

#### **pk 集合 - 添加 robotBinding 字段**（群聊绑定）
```javascript
{
  // ... 原有字段 ...
  robotBinding: {
    gid: 'xxx@chatroom',          // 群ID (gid)，绑定后存储，解绑后清空
    wechatMid: 'wxid_xxx'         // 发送绑定消息的群成员ID（mid）
  }
}
```

#### **robotConfigs 集合 - 机器人配置集合**（统一存储所有配置信息）
```javascript
{
  _id: 'xxx',
  type: 'friend',                  // 'friend'（好友）或 'group'（群聊）
  targetId: 'wxid_xxx',           // 好友ID（mid）或群ID（gid）
  openId: 'user_openid',          // 小程序用户openId（好友绑定）
  pkId: 'pk_xxx',                 // PK赛ID（群聊绑定）
  
  status: 'active',               // 'active'/'inactive'
  speechStyle: 'encourage',       // 'sarcastic', 'cute', 'encourage', 'professional', 'custom'
  customSpeechTemplates: [],      // 自定义话术模板
  
  config: {
    // 打卡触发配置
    checkInTrigger: {
      enabled: true,
      delayMinutes: 0,
      personalCheckIn: { enabled: true },      // 个人绑定使用
      pkCheckIn: { enabled: true, atUser: true } // 群聊绑定使用
    },
    // 定时消息配置
    scheduledMessages: {
      morningPush: { enabled: true, timeRange: '07:30-08:30' },
      lunchReminder: { enabled: true, timeRange: '11:30-12:30' },
      eveningReminder: { enabled: true, timeRange: '19:00-20:00' },
      dailySummary: { enabled: true, timeRange: '21:00-22:00' },  // 个人绑定使用
      dailyReport: { enabled: true, timeRange: '21:00-22:00' }     // 群聊绑定使用
    },
    // 关键词配置
    keywords: {}
  },
  
  // 打卡触发记录（用于去重）
  lastCheckInCheck: {
    date: '2025-01-20',           // 上次检查日期
    checked: false,               // 今日是否已发送提醒（个人绑定）
    checkedUsers: ['openid1', 'openid2'], // 已发送提醒的用户openId列表（群聊绑定）
    lastCheckTime: Date           // 上次检查时间
  },
  
  createdAt: Date,
  updatedAt: Date
}
```

#### **robotMessages** - 消息记录表
```javascript
{
  _id: 'xxx',
  bindingId: 'binding_xxx',     // 关联的绑定关系ID
  type: 'group',                 // 'group' 或 'friend'
  targetId: 'xxx@chatroom',      // 群ID或好友ID
  messageType: 'check_in',       // 消息类型
  triggerType: 'check_in',       // 触发类型
  triggerUserId: 'openid_xxx',    // 触发用户
  content: '消息内容',
  speechStyle: 'encourage',      // 使用的话术类型
  createdAt: Date
}
```

### 4.2 关联关系

```
users
    └── mid → 微信用户ID（好友绑定）

pk.robotBinding
    ├── gid → 群ID（群聊绑定）
    └── wechatMid → 发送绑定消息的群成员ID

robotConfigs
    ├── type='friend' + targetId → users.mid（好友配置）
    └── type='group' + targetId → pk.robotBinding.gid（群聊配置）

robotMessages
    ├── type: 'friend' → users.openId（个人消息）
    └── type: 'group' → pk._id（群聊消息）

打卡触发查询：
    pk.robotBinding.gid → robotConfigs → pk.members → records（PK打卡）
    users.mid → robotConfigs → records（个人打卡）
```

## 五、云函数架构

### 5.1 wechatBotNew 云函数结构

```javascript
exports.main = async (event, context) => {
  // 1. 解析参数
  const { gid, wxuin, skw, content, mid } = parseParams(event);
  
  // 2. 判断触发类型
  if (content && content.includes('绑定：')) {
    // 绑定处理
    return await handleBind(content, gid, wxuin, mid, skw);
  }
  
  if (content) {
    // 关键词触发
    return await handleKeyword(content, gid, wxuin, skw);
  }
  
  // 定时触发（主动消息接口）
  // 3. 先检查打卡触发
  const checkInMsg = await checkCheckInTrigger(gid, wxuin, skw);
  if (checkInMsg) return checkInMsg;
  
  // 4. 再检查定时推送
  return await handleScheduledMessage(gid, wxuin, skw);
}
```

## 十、安全与性能

### 10.1 安全性
- **数据校验**：验证机器人平台请求参数
- **权限控制**：区分管理员和普通用户权限
- **数据隔离**：不同用户的绑定数据隔离

### 10.2 性能优化
- **配置缓存**：机器人配置缓存到内存
- **消息去重**：避免重复推送
- **数据库优化**：合理使用索引，避免全表扫描
- **异步处理**：打卡检测异步处理

### 10.3 容错处理
- **机器人离线**：优雅处理离线状态
- **API失败**：提供降级方案
- **数据异常**：数据校验和默认值

## 十一、总结

### 11.1 架构特点

1. **模块化设计**：功能模块清晰，易于维护和扩展
2. **统一入口**：一个云函数处理所有请求，统一管理
3. **数据驱动**：通过配置数据控制机器人行为
4. **角色分离**：管理员和普通用户功能分离

### 11.2 关键设计决策

1. **一个机器人应用**：简化管理，降低复杂度
2. **绑定消息格式**：群聊用PK ID，好友用用户ID
3. **打卡触发优先**：先检查打卡触发，再检查定时推送
4. **话术类型统一**：所有消息使用统一的话术类型配置

### 11.3 扩展性

- ✅ 支持添加新的话术类型
- ✅ 支持添加新的关键词
- ✅ 支持自定义定时规则
- ✅ 支持多机器人管理（未来扩展）
