# 微信机器人技术设计方案

## 一、数据库设计

### 1.1 修改现有集合

#### **users 集合 - 添加机器人绑定字段**

**新增字段：**
```javascript
{
  // ... 原有字段 ...
  
  // 微信用户ID（mid），机器人绑定后存储
  mid: 'wxid_xxx'
}
```

**说明：**
- `mid`: 微信用户ID（mid/wxuin），绑定后存储，解绑后清空为 `null`
- 配置信息存储在独立的 `robotConfigs` 集合中

#### **pk 集合 - 添加机器人绑定字段**

**新增字段：**
```javascript
{
  // ... 原有字段 ...
  
  // 机器人绑定信息（群聊绑定）
  robotBinding: {
    gid: 'xxx@chatroom',          // 群ID (gid)
    wechatMid: 'wxid_xxx'         // 发送绑定消息的群成员ID（mid）
  }
}
```

**说明：**
- `robotBinding.gid`: 群ID，绑定后存储，解绑后清空为 `null`
- `robotBinding.wechatMid`: 发送绑定消息的群成员ID（mid）
- 配置信息存储在独立的 `robotConfigs` 集合中

**注意：**
- 好友绑定：`users.mid` 存储微信用户ID（mid/wxuin）
- 群聊绑定：`pk.robotBinding.gid` 存储群ID，`pk.robotBinding.wechatMid` 存储发送绑定消息的群成员ID（mid）
- 所有配置信息统一存储在 `robotConfigs` 集合中，通过 `targetId` 和 `type` 关联

### 1.2 新增集合

#### **robotConfigs** - 机器人配置集合（统一存储所有配置信息）
```javascript
{
  _id: 'xxx',
  type: 'friend',                  // 'friend'（好友）或 'group'（群聊）
  targetId: 'wxid_xxx',           // 好友ID（mid）或群ID（gid）
  openId: 'user_openid',          // 小程序用户openId（好友绑定）
  pkId: 'pk_xxx',                 // PK赛ID（群聊绑定）
  
  status: 'active',               // 'active'/'inactive'
  speechStyle: 'encourage',       // 'sarcastic', 'cute', 'encourage', 'professional', 'custom'
  customSpeechTemplates: [],      // 自定义话术模板
  
  config: {
    // 打卡触发配置
    checkInTrigger: {
      enabled: true,
      delayMinutes: 0,
      personalCheckIn: { enabled: true },      // 个人绑定使用
      pkCheckIn: { enabled: true, atUser: true } // 群聊绑定使用
    },
    // 定时消息配置
    scheduledMessages: {
      morningPush: { enabled: true, timeRange: '07:30-08:30' },
      lunchReminder: { enabled: true, timeRange: '11:30-12:30' },
      eveningReminder: { enabled: true, timeRange: '19:00-20:00' },
      dailySummary: { enabled: true, timeRange: '21:00-22:00' },  // 个人绑定使用
      dailyReport: { enabled: true, timeRange: '21:00-22:00' }     // 群聊绑定使用
    },
    // 关键词配置
    keywords: {}
  },
  
  // 打卡触发记录（用于去重）
  lastCheckInCheck: {
    date: '2025-01-20',           // 上次检查日期
    checked: false,               // 今日是否已发送提醒（个人绑定）
    checkedUsers: ['openid1', 'openid2'], // 已发送提醒的用户openId列表（群聊绑定）
    lastCheckTime: Date           // 上次检查时间
  },
  
  createdAt: Date,
  updatedAt: Date
}
```

**关联关系：**
- 好友绑定：`robotConfigs.targetId` = `users.mid`，`robotConfigs.type` = 'friend'
- 群聊绑定：`robotConfigs.targetId` = `pk.robotBinding.gid`，`robotConfigs.type` = 'group'，`robotConfigs.pkId` = `pk._id`

#### **robotMessages** - 机器人消息记录
```javascript
{
  _id: 'xxx',
  type: 'group',                  // 'group' 或 'friend'
  targetId: 'xxx@chatroom',       // 群ID或好友ID
  pkId: 'pk_xxx',                 // PK赛ID（仅群聊）
  openId: 'user_openid',          // 用户openId（仅好友）
  messageType: 'morning',         // 消息类型：morning/daytime/evening/daily_report/check_in/keyword
  triggerType: 'scheduled',       // 触发类型：scheduled/keyword/check_in
  triggerUserId: 'openid_xxx',    // 触发用户（打卡触发时使用）
  content: '消息内容',
  imageUrl: '',                   // 图片URL（如果有）
  speechStyle: 'encourage',       // 使用的话术类型
  createdAt: Date
}
```

**注意：**
- `robotConfigs` 集合统一存储所有机器人配置信息
- `robotMessages` 中的 `targetId` 用于关联群聊或好友，`pkId` 或 `openId` 用于关联具体的PK赛或用户
- 查询配置时：通过 `targetId` + `type` 查询 `robotConfigs` 集合

## 二、云函数设计

### 2.1 增强现有云函数 `wechatBotNew`

**功能扩展：**
1. 支持群聊和好友两种类型（通过 `gid` 或 `wxuin` 区分）
2. 支持自定义配置（从数据库读取配置）
3. **支持打卡触发检测**（检查用户是否打卡，立即发送话术）
4. **支持话术类型**（根据配置的话术类型生成消息）
5. **支持定时推送**（群聊和好友都支持）
6. 支持消息记录（保存到 `robotMessages` 集合）
7. 根据绑定类型决定可用功能（好友不支持关联PK赛）

**核心逻辑流程：**
```javascript
// 1. 解析参数
const { gid, wxuin, skw, content, mid, nickname } = bodyParams;
const targetId = gid || wxuin || mid; // 群聊用gid，好友用wxuin或mid
const type = gid ? 'group' : 'friend';

// 2. 处理绑定关键词（"绑定：PK ID"或"绑定：用户ID"）
if (content && content.includes('绑定：')) {
  return await handleBindKeyword(content, gid, mid, skw, type);
}

// 3. 查询绑定信息和配置
let binding = null;
let config = null;

if (type === 'group') {
  // 群聊：通过gid查询pk集合和robotConfigs集合
  const pkRes = await db.collection('pk').where({
    'robotBinding.gid': gid
  }).get();
  if (pkRes.data.length > 0) {
    const pkInfo = pkRes.data[0];
    // 查询配置信息
    const configRes = await db.collection('robotConfigs').where({
      targetId: gid,
      type: 'group',
      status: 'active'
    }).get();
    if (configRes.data.length > 0) {
      config = configRes.data[0];
      binding = {
        gid: pkInfo.robotBinding.gid,
        wechatMid: pkInfo.robotBinding.wechatMid,
        pkId: pkInfo._id,
        openId: pkInfo.openId,
        ...config
      };
    }
  }
} else {
  // 好友：通过mid查询users集合和robotConfigs集合
  const userRes = await db.collection('users').where({
    mid: mid
  }).get();
  if (userRes.data.length > 0) {
    const userInfo = userRes.data[0];
    // 查询配置信息
    const configRes = await db.collection('robotConfigs').where({
      targetId: mid,
      type: 'friend',
      status: 'active'
    }).get();
    if (configRes.data.length > 0) {
      config = configRes.data[0];
      binding = {
        mid: userInfo.mid,
        openId: userInfo.openId,
        ...config
      };
    }
  }
}

if (!binding) {
  return createEmptyMessage(); // 未绑定
}

// 4. 判断触发类型
if (content) {
  // 关键词触发（群聊和好友都支持）
  return handleKeywordTrigger(binding, content, type);
} else {
  // 定时触发（主动消息接口）
  // 先检查打卡触发，再检查定时推送
  const checkInMessage = await checkCheckInTrigger(binding, type);
  if (checkInMessage) {
    // 好友打卡评价需要返回wxid
    if (type === 'friend') {
      return JSON.stringify({
        rs: 1,
        wxid: binding.mid, // 好友ID
        tip: checkInMessage.tip,
        end: 1
      });
    }
    return JSON.stringify(checkInMessage);
  }
  // 定时推送
  const scheduledMessage = await handleScheduledMessage(binding, type);
  // 好友定时推送需要返回wxid
  if (type === 'friend' && scheduledMessage) {
    return JSON.stringify({
      rs: 1,
      wxid: binding.mid, // 好友ID
      tip: scheduledMessage.tip,
      end: 1
    });
  }
  return scheduledMessage ? JSON.stringify(scheduledMessage) : createEmptyMessage();
}
```

**打卡触发检测逻辑：**
```javascript
async function checkCheckInTrigger(binding, type) {
  const config = binding.config?.checkInTrigger;
  if (!config || !config.enabled) {
    return null;
  }
  
  const today = getTodayDate();
  const lastCheck = binding.lastCheckInCheck || {};
  
  // 检查是否需要检查（避免频繁检查）
  const lastCheckTime = lastCheck.lastCheckTime ? new Date(lastCheck.lastCheckTime).getTime() : 0;
  if (lastCheck.date === today && 
      Date.now() - lastCheckTime < 60000) { // 1分钟内不重复检查
    return null;
  }
  
  if (type === 'friend') {
    // 个人打卡评价
    return await checkPersonalCheckIn(binding, config);
  } else {
    // PK打卡评价
    return await checkPkCheckIn(binding, config);
  }
}

async function checkPersonalCheckIn(binding, config) {
  // 查询用户今日打卡记录
  const records = await db.collection('records').where({
    openId: binding.openId,
    date: getTodayDate()
  }).get();
  
  if (records.data.length === 0) {
    return null; // 未打卡
  }
  
  // 检查是否已发送过评价
  const lastCheck = binding.lastCheckInCheck || {};
  if (lastCheck.date === getTodayDate() && lastCheck.checked) {
    return null; // 已发送过
  }
  
  // 生成话术
  const message = generateCheckInMessage(
    binding.speechStyle,
    records.data[0],
    'personal'
  );
  
  // 更新检查记录（更新robotConfigs集合）
  await db.collection('robotConfigs').where({
    _id: binding._id
  }).update({
    data: {
      'lastCheckInCheck': {
        date: getTodayDate(),
        checked: true,
        lastCheckTime: db.serverDate()
      },
      updatedAt: db.serverDate()
    }
  });
  
  return message;
}

async function checkPkCheckIn(binding, config) {
  if (!binding.pkId) {
    return null;
  }
  
  // 查询PK成员
  const pk = await db.collection('pk').where({ _id: binding.pkId }).get();
  const members = pk.data[0]?.members || [];
  const openIds = members.map(m => m.openId);
  
  // 查询今日打卡记录
  const records = await db.collection('records').where({
    openId: db.command.in(openIds),
    date: getTodayDate()
  }).get();
  
  // 找出新打卡的用户（对比上次检查）
  const lastChecked = binding.lastCheckInCheck?.checkedUsers || [];
  const newCheckIns = records.data.filter(r => 
    !lastChecked.includes(r.openId)
  );
  
  if (newCheckIns.length === 0) {
    return null;
  }
  
  // 生成话术（可@多个用户）
  const message = generatePkCheckInMessage(
    binding.speechStyle,
    newCheckIns,
    members,
    config.pkCheckIn.atUser
  );
  
  // 更新检查记录（更新robotConfigs集合）
  await db.collection('robotConfigs').where({
    _id: binding._id
  }).update({
    data: {
      'lastCheckInCheck': {
        date: getTodayDate(),
        checkedUsers: [...lastChecked, ...newCheckIns.map(r => r.openId)],
        lastCheckTime: db.serverDate()
      },
      updatedAt: db.serverDate()
    }
  });
  
  return message;
}
```

```javascript
function generateCheckInMessage(speechStyle, record, type) {
  const user = record.userInfo || {};
  const weight = record.weight;
  const bmi = calculateBMI(weight, user.height);
  
  const templates = {
    sarcastic: [
      `今天打卡了？体重${weight}kg，${bmi > 24 ? '还是有点重哦' : '还不错'}！继续坚持！`,
      `打卡成功！${weight}kg，离目标还有距离，别松懈！`
    ],
    cute: [

      `打卡成功！当前${weight}kg，你真的很努力呢～`
    ],
    encourage: [
      `恭喜完成今日打卡！体重${weight}kg，BMI：${bmi}，坚持就是胜利！`,
      `打卡成功！当前${weight}kg，你已经很棒了，继续保持！`
    ],
    professional: [
      `打卡记录：体重${weight}kg，BMI：${bmi}，${bmi > 24 ? '建议继续减重' : '指标正常'}。`,
      `今日数据：体重${weight}kg，BMI：${bmi}，请继续保持。`
    ]
  };
  
  const styleTemplates = templates[speechStyle] || templates.encourage;
  const message = styleTemplates[Math.floor(Math.random() * styleTemplates.length)];
  
  return {
    rs: 1,
    tip: message,
    end: 0
  };
}
```

**绑定关键词处理逻辑：**
```javascript
async function handleBindKeyword(content, gid, mid, skw, type) {
  // 解析"绑定：PK ID"或"绑定：用户ID"格式
  const match = content.match(/绑定[：:]\s*(\w+)/);
  if (!match) {
    return createMessage("绑定格式错误，请发送：绑定：PK ID（群聊）或绑定：用户ID（好友）");
  }
  
  const bindId = match[1]; // 提取ID（群聊是PK ID，好友是用户ID）
  const wechatMid = mid; // 微信用户ID（群聊中是群成员ID，好友中是好友ID）
  
  if (type === 'group') {
    // 群聊绑定 - 使用PK ID
    // 1. 通过PK ID查询PK赛信息
    const pkRes = await db.collection('pk').where({ _id: bindId }).get();
    if (pkRes.data.length === 0) {
      return createMessage("未找到PK赛信息，请检查PK ID是否正确");
    }
    
    const pkInfo = pkRes.data[0];
    
    // 2. 更新PK集合的robotBinding字段（只存储绑定信息）
    await db.collection('pk').where({ _id: bindId }).update({
      data: {
        robotBinding: {
          gid: gid,
          wechatMid: wechatMid
        }
      }
    });
    
    // 3. 创建或更新robotConfigs配置记录
    const configRes = await db.collection('robotConfigs').where({
      targetId: gid,
      type: 'group'
    }).get();
    
    const configData = {
      type: 'group',
      targetId: gid,
      pkId: bindId,
      openId: pkInfo.openId,
      status: 'active',
      speechStyle: 'encourage',
      customSpeechTemplates: [],
      config: {
        checkInTrigger: {
          enabled: true,
          delayMinutes: 0,
          pkCheckIn: { enabled: true, atUser: true }
        },
        scheduledMessages: {
          morningPush: { enabled: true, timeRange: '07:30-08:30' },
          lunchReminder: { enabled: true, timeRange: '11:30-12:30' },
          eveningReminder: { enabled: true, timeRange: '19:00-20:00' },
          dailyReport: { enabled: true, timeRange: '21:00-22:00' }
        },
        keywords: {}
      },
      lastCheckInCheck: {
        date: '',
        checkedUsers: [],
        lastCheckTime: null
      },
      updatedAt: db.serverDate()
    };
    
    if (configRes.data.length > 0) {
      // 更新现有配置
      await db.collection('robotConfigs').where({
        _id: configRes.data[0]._id
      }).update({
        data: {
          ...configData,
          updatedAt: db.serverDate()
        }
      });
    } else {
      // 创建新配置
      await db.collection('robotConfigs').add({
        data: {
          ...configData,
          createdAt: db.serverDate()
        }
      });
    }
    
    // 3. 更新PK成员信息，将mid关联到对应的用户（如果成员中有匹配的）
    // 通过mid找到对应的PK成员，更新wechatMid
    // 这里需要根据实际情况实现匹配逻辑（可能需要通过其他方式建立关联）
    
    return createMessage("绑定成功！群聊已关联到PK赛。");
  } else {
    // 好友绑定 - 使用用户ID
    // 1. 通过用户ID查询用户信息（假设用户ID就是openId，或需要建立映射关系）
    const userRes = await db.collection('users').where({
      // 这里需要根据实际的用户ID格式查询
      // 可能是openId，也可能是其他唯一标识
      _id: bindId
    }).get();
    
    if (userRes.data.length === 0) {
      return createMessage("未找到用户信息，请检查用户ID是否正确");
    }
    
    const userInfo = userRes.data[0];
    
    // 2. 更新用户集合的mid字段
    await db.collection('users').where({
      _id: userInfo._id
    }).update({
      data: {
        mid: wechatMid,              // 微信用户ID
        updatedAt: db.serverDate()
      }
    });
    
    // 3. 创建或更新robotConfigs配置记录
    const configRes = await db.collection('robotConfigs').where({
      targetId: wechatMid,
      type: 'friend'
    }).get();
    
    const configData = {
      type: 'friend',
      targetId: wechatMid,
      openId: userInfo.openId,
      status: 'active',
      speechStyle: 'encourage',
      customSpeechTemplates: [],
      config: {
        checkInTrigger: {
          enabled: true,
          delayMinutes: 0,
          personalCheckIn: { enabled: true }
        },
        scheduledMessages: {
          morningPush: { enabled: true, timeRange: '07:30-08:30' },
          lunchReminder: { enabled: true, timeRange: '11:30-12:30' },
          eveningReminder: { enabled: true, timeRange: '19:00-20:00' },
          dailySummary: { enabled: true, timeRange: '21:00-22:00' }
        },
        keywords: {}
      },
      lastCheckInCheck: {
        date: '',
        checked: false,
        lastCheckTime: null
      },
      updatedAt: db.serverDate()
    };
    
    if (configRes.data.length > 0) {
      // 更新现有配置
      await db.collection('robotConfigs').where({
        _id: configRes.data[0]._id
      }).update({
        data: {
          ...configData,
          updatedAt: db.serverDate()
        }
      });
    } else {
      // 创建新配置
      await db.collection('robotConfigs').add({
        data: {
          ...configData,
          createdAt: db.serverDate()
        }
      });
    }
    
    return createMessage("绑定成功！好友ID已关联到你的账号。");
  }
}
```

### 3.2 机器人平台调用云函数

**接收地址配置：**
```
https://your-domain.com/wechatBotNew
或
云函数HTTP触发地址
```

**接口类型：**

1. **群聊关键词触发**（[群]关键词触发接口）
   - 参数：`gid`, `skw`, `content` 等
   - 返回：`{"rs":1,"tip":"消息内容","end":0}`

2. **群聊定时推送**（[群]主动消息接口）
   - 参数：`gid`, `skw`
   - 返回：`{"rs":1,"tip":"消息内容","end":1}`

3. **好友关键词触发**（[友]关键词触发接口）
   - 参数：`wxuin`, `skw`, `content` 等
   - 返回：`{"rs":1,"tip":"消息内容","end":0}`

4. **好友/群定时推送**（[友/群]主动消息接口）
   - 参数：`skw`（通过skw查找所有绑定关系）
   - 返回：`{"rs":1,"wxid":"wxid_xxx或xxx@chatroom","tip":"消息内容","end":1}`
   - **注意**：此接口需要返回 `wxid` 字段指定发送对象

**参数处理：**
- 机器人平台通过 POST 方式发送参数
- 云函数解析 body 参数
- 根据 `gid`/`wxuin` 和 `skw` 查找绑定信息
- 根据绑定类型返回相应格式的消息

## 五、打卡触发机制详解

### 5.1 触发时机

**机器人平台定时请求：**
- 稳定版：每2分钟请求一次
- 开发版：每60秒请求一次

**检测逻辑：**
1. 云函数每次被调用时，先检查打卡触发
2. 对比上次检查时间和已发送提醒的用户列表
3. 找出新打卡的用户
4. 生成话术并返回
5. 更新检查记录，避免重复发送

### 5.2 去重机制

**防止重复发送：**
- 记录每日已发送提醒的用户列表
- 同一用户同一天只发送一次打卡评价
- 使用 `lastCheckInCheck` 字段记录检查状态

**数据结构：**
```javascript
lastCheckInCheck: {
  date: '2025-01-20', // 检查日期
  checkedUsers: ['openid1', 'openid2'], // 已发送的用户
  lastCheckTime: Date // 上次检查时间戳
}
```

### 5.3 延迟触发

**配置选项：**
- 立即触发：`delayMinutes: 0`
- 延迟5分钟：`delayMinutes: 5`
- 延迟10分钟：`delayMinutes: 10`

**实现方式：**
- 记录打卡时间
- 在定时触发时检查是否达到延迟时间
- 达到延迟时间后才发送提醒

## 六、数据流转图

```
用户在小程序打卡
    ↓

    ↓

    ↓

    ├─ 查询绑定关系

    ↓
返回消息给机器人平台
    ↓
机器人发送消息到群/好友
    ↓
保存消息记录到 robotMessages
```

## 七、错误处理

### 7.1 常见错误场景

1. **绑定关系不存在**
   - 返回空消息，不中断流程
   - 记录错误日志

2. **数据库查询失败**
   - 返回默认消息
   - 记录错误日志

3. **话术生成失败**
   - 使用默认话术模板
   - 记录错误日志

4. **机器人平台请求异常**
   - 云函数正常返回，由平台重试
   - 记录请求日志

### 7.2 日志记录

**需要记录的关键信息：**
- 请求参数（gid/wxuin, skw, content等）
- 绑定关系查询结果
- 打卡记录查询结果
- 生成的消息内容
- 错误信息（如有）
