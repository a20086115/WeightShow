# 微信机器人产品设计方案

## 一、产品定位与架构

### 1.1 产品定位

将微信机器人功能集成到"瘦身打卡助手"小程序中，**使用一个微信机器人账号**，服务多个群聊和好友，实现：
- **打卡触发提醒**：用户打卡后立即发送个性化话术（核心功能）
- **定时推送**：定时推送打卡提醒、减肥技巧、战队战报
- **社交激励**：通过群聊增强用户粘性和打卡动力

### 1.2 核心架构

```
一个微信机器人账号（一个skw应用）
    ├── 绑定1：PK群聊A（关联PK赛1，配置A）
    ├── 绑定2：PK群聊B（关联PK赛2，配置B）
    ├── 绑定3：个人好友C（配置C）
    └── 绑定4：个人好友D（配置D）
```

**核心概念：**
- ✅ **一个机器人账号**：统一管理，所有绑定关系共享同一个机器人应用（skw）
- ✅ **多个绑定关系**：可以绑定多个群聊/好友，每个绑定独立配置
- ✅ **独立配置**：每个群/好友有独立的配置（话术类型、定时推送、关键词等）
- ✅ **类型区分**：群聊绑定关联PK赛，好友绑定关联个人账号

### 1.3 用户角色

- **管理员**：拥有机器人管理权限，可以配置所有绑定关系
- **普通用户**：可以绑定个人好友，可以切换话术类型

## 三、模块一：绑定功能

### 3.1 个人绑定（好友）

#### **功能说明**
用户绑定个人微信好友，机器人会向该好友发送打卡评价和定时消息。

#### **绑定流程**
```
1. 进入"我的" → 点击"微信机器人"
2. 系统判断绑定状态：
   - 未绑定：显示功能列表和绑定引导界面
   - 已绑定：显示功能列表、解绑、话术切换界面
3. 未绑定状态下，用户操作：
   - 查看机器人功能介绍（打卡评价、定时推送、个性化话术）
   - 点击"一键复制绑定消息"按钮
   - 绑定消息自动生成：绑定：{用户ID}（如：绑定：user_12345）
   - 添加机器人为好友（如果还没添加）
   - 给机器人发送复制的绑定消息
4. 机器人收到消息后：
   - 机器人通过webhook接口收到消息（包含mid）
   - 云函数解析消息，识别"绑定：用户ID"关键词
   - 通过用户ID查询用户信息（openId）
   - 将mid（好友ID）存储到users集合的mid字段，配置信息存储到robotConfig字段
   - 绑定完成，小程序页面自动刷新，显示已绑定状态界面
5. 已绑定状态下，用户可操作：
   - 查看所有功能状态
   - 开启/关闭各项功能
   - 切换话术类型
   - 解绑机器人
```

#### **页面设计**
**位置**：`pages/robot/index`（机器人主界面）

**功能说明：**
根据用户绑定状态显示不同内容：
- **未绑定**：展示机器人功能，引导绑定
- **已绑定**：展示功能列表、解绑、话术切换

### 3.2 PK群绑定（群聊）

#### **功能说明**
在PK详情页面绑定群聊，机器人会向该群发送PK打卡评价和定时消息。

#### **绑定流程**
```
1. 进入PK详情页面（pages/pk/pk）
2. 点击"绑定机器人"按钮
3. 弹出绑定引导弹窗，显示：
   - 绑定步骤说明
   - 绑定消息：绑定：{PK ID}
   - "一键复制"按钮
4. 用户操作：
   - 点击"一键复制"按钮
   - 在PK群内粘贴并发送给机器人
   - 确保机器人已在群内（如果不在，先邀请机器人入群）
5. 机器人收到消息后：
   - 机器人通过webhook接口收到消息（包含gid和mid）
   - 云函数解析消息，识别"绑定：PK ID"关键词
   - 通过PK ID查询PK赛信息，获取所有PK成员
   - 将gid（群ID）和pkId（PK赛ID）存储到pk集合的robotBinding字段
   - 将mid（发送消息的群成员ID）关联到对应的用户表
   - 绑定完成，小程序页面自动更新绑定状态
6. 绑定完成后：
   - 弹窗自动关闭
   - "绑定机器人"按钮变为"机器人管理"按钮
```

### 3.3 绑定数据存储

**绑定消息格式：**
- **群聊绑定**：`绑定：{PK ID}` - 通过PK ID关联整个PK赛
- **好友绑定**：`绑定：{用户ID}` - 通过用户ID关联个人账号

**数据存储：**
- **群聊绑定**：
  - 在 `pk` 集合中添加 `robotBinding` 字段，存储 `gid`（群ID）和 `wechatMid`（发送绑定消息的群成员ID）
  - 在 `robotConfigs` 集合中创建配置记录，`targetId` = `gid`，`type` = 'group'，`pkId` = PK赛ID
  - 通过PK ID查询所有PK成员，将 `mid`（发送绑定消息的群成员ID）关联到对应的用户表
- **好友绑定**：
  - 在 `users` 集合中添加 `mid` 字段，存储微信用户ID（mid/wxuin）
  - 在 `robotConfigs` 集合中创建配置记录，`targetId` = `mid`，`type` = 'friend'，`openId` = 用户openId
  - 所有配置信息（话术类型、打卡评价配置、定时推送配置等）统一存储在 `robotConfigs` 集合中

## 四、模块二：打卡评价功能（核心）

### 4.1 功能说明

**打卡评价**与**提醒打卡**是两种不同的功能：
- **打卡评价**：用户完成打卡后，机器人检测到打卡行为，立即发送个性化评价/反馈消息
- **提醒打卡**：定时推送功能中的"晚上打卡提醒推送"，提醒用户去打卡（见模块三）

### 4.2 个人打卡评价（好友）

#### **功能说明**
用户在小程序完成打卡后，机器人定时触发时检测到打卡，立即发送个性化评价话术给用户。

#### **触发流程**
```
1. 用户在小程序完成打卡（插入records记录）
2. 机器人平台定时请求云函数（每2分钟）
3. 云函数检查：
   - 查询该好友绑定关系
   - 检查是否开启"个人打卡评价"
   - 查询用户今日是否有新打卡记录
   - 检查是否已发送过打卡评价（避免重复）
4. 如果发现新打卡：
   - 根据话术类型生成评价话术
   - 获取用户打卡数据（体重、BMI等）
   - 生成个性化评价消息
5. 返回消息给机器人平台（包含wxid）
6. 机器人发送评价消息给用户（好友）
```

#### **配置项**
- ✅ 开启/关闭个人打卡评价
- ✅ 触发延迟时间（立即/5分钟后/10分钟后）
- ✅ 话术类型选择

### 4.3 PK打卡评价（群聊）

#### **功能说明**
PK成员在小程序完成打卡后，机器人定时触发时检测到打卡，立即在群里发送评价话术。

#### **触发流程**
```
1. PK成员在小程序完成打卡（插入records记录）
2. 机器人平台定时请求云函数（每2分钟）
3. 云函数检查：
   - 查询该群聊绑定关系
   - 检查是否开启"PK打卡评价"
   - 检查关联的PK赛
   - 查询PK成员今日打卡记录
   - 找出新打卡的成员（对比上次检查时间）
4. 如果发现新打卡：
   - 根据话术类型生成评价话术
   - 获取打卡用户信息、打卡数据
   - 生成个性化评价消息（可@用户）
5. 返回消息给机器人平台
6. 机器人发送评价消息到群内
```

#### **配置项**
- ✅ 开启/关闭PK打卡评价
- ✅ 是否@打卡用户
- ✅ 触发延迟时间（立即/5分钟后/10分钟后）
- ✅ 话术类型选择

### 4.4 去重机制

**防止重复发送：**
- 记录每日已发送评价的用户列表
- 同一用户同一天只发送一次打卡评价
- 使用 `lastCheckInCheck` 字段记录检查状态

## 五、模块三：定时推送功能

### 5.1 功能说明

机器人按照预设时间自动推送消息，**针对用户（好友）和群聊设计不同的推送内容和逻辑**。

### 5.2 用户（好友）定时推送设计

#### **5.2.1 早上激励推送**
**推送时间**：7:30-8:30（可配置）
**推送频率**：每天一次
**推送内容**：
- 今日减肥口号/激励话语
- 根据话术类型生成个性化内容
- 可包含用户当前体重、目标体重、BMI等信息

**内容示例：**
- 毒嘴型：`早上好！今天也要努力减肥哦，别让昨天的努力白费！当前体重{weight}kg，距离目标还有{loss}kg，加油！`
- 可爱型：`早上好呀～新的一天开始了，今天也要坚持打卡哦！当前{weight}kg，目标{aimWeight}kg，我们一起加油💪`
- 鼓励型：`早上好！新的一天，新的开始！坚持就是胜利，你已经很棒了！当前体重{weight}kg，继续加油！`
- 专业型：`早上好。今日数据：当前体重{weight}kg，BMI：{bmi}，目标体重{aimWeight}kg，还需减重{loss}kg。`

**触发条件**：
- 绑定关系存在且开启定时推送
- 当前时间在推送时间范围内
- 今日未推送过该类型消息

#### **5.2.2 中午饮食提醒推送**
**推送时间**：11:30-12:30（可配置）
**推送频率**：每天一次
**推送内容**：
- 减肥饮食小知识
- 午餐建议
- 可结合用户当前体重情况给出建议

**内容示例：**
- 毒嘴型：`中午了，该吃饭了！记住：少油少盐，八分饱就够了。别又吃多了，不然今天的努力就白费了！`
- 可爱型：`中午好～该吃午餐啦！建议：少油少盐，八分饱更健康哦～记得多喝水💧`
- 鼓励型：`中午好！午餐时间到了，记得选择健康的食物，控制食量，八分饱即可。坚持就是胜利！`
- 专业型：`午餐建议：控制热量摄入，建议选择低脂高蛋白食物，控制食量在八分饱。`

**触发条件**：
- 绑定关系存在且开启定时推送
- 当前时间在推送时间范围内
- 今日未推送过该类型消息

#### **5.2.3 晚上打卡提醒推送**
**推送时间**：19:00-20:00（可配置）

### 5.4 推送流程设计

#### **5.4.1 用户（好友）推送流程**
```
1. 机器人平台定时请求云函数（每2分钟）
   - 使用[友/群]主动消息接口
2. 云函数处理：
   - 查询所有好友绑定关系（status='active'）
   - 遍历每个绑定关系：
     a. 判断当前时间，决定推送类型
     b. 检查是否开启该时段推送
     c. 检查今日是否已推送过（去重）
     d. 查询用户数据（体重、BMI、打卡记录等）
     e. 根据话术类型生成个性化消息
     f. 返回消息（包含wxid）
3. 机器人平台发送消息给对应好友
4. 保存消息记录到robotMessages
```

#### **5.4.2 群聊（PK）推送流程**
```
1. 机器人平台定时请求云函数（每2分钟）
   - 使用[群]主动消息接口
2. 云函数处理：
   - 查询所有群聊绑定关系（status='active'）
   - 遍历每个绑定关系：
     a. 判断当前时间，决定推送类型
     b. 检查是否开启该时段推送
     c. 检查今日是否已推送过（去重）
     d. 查询PK赛数据（成员、打卡记录等）
     e. 计算统计数据（打卡率、减重等）
     f. 根据话术类型生成消息
     g. 返回消息
3. 机器人平台发送消息到群内
4. 保存消息记录到robotMessages
```

### 5.6 推送内容变量

#### **用户（好友）可用变量**
- `{weight}` - 当前体重（kg）
- `{aimWeight}` - 目标体重（kg）
- `{loss}` - 还需减重量（kg）
- `{bmi}` - BMI值
- `{hasCheckIn}` - 今日是否已打卡（true/false）
- `{weekDays}` - 本周打卡天数
- `{monthLoss}` - 本月累计减重（kg）
- `{continuousDays}` - 连续打卡天数

#### **群聊（PK）可用变量**
- `{completedCount}` - 今日已打卡人数
- `{totalCount}` - PK总人数
- `{uncompletedCount}` - 今日未打卡人数
- `{memberList}` - 成员打卡列表（格式化字符串）
- `{totalLoss}` - 战队累计减重（kg）
- `{topLoser}` - 减重最多成员信息
- `{completionRate}` - 打卡率（%）

### 5.7 推送去重机制

**防止重复推送：**
- 记录每日已推送的消息类型
- 同一类型消息同一天只推送一次
- 使用 `pushRecords` 集合记录推送历史

**数据结构：**
```javascript
pushRecords: {
  date: '2025-01-20',
  messages: {
    morningPush: true,      // 已推送早上推送
    lunchReminder: true,    // 已推送中午提醒
    eveningReminder: true,  // 已推送晚上提醒
    dailySummary: true      // 已推送每日总结
  },
  lastPushTime: Date
}
```

```

```

### 8.5 解绑流程

```

   - type=friend: 清除users集合中对应openId的robotBinding字段
   - type=group: 清除pk集合中对应pkId的robotBinding字段
4. 解绑成功后提示"已解绑"
5. 返回上一页，更新页面状态
```

## 九、模块七：话术类型切换（普通用户）

### 9.1 功能说明

普通用户可以切换机器人话术类型，个性化机器人回复风格。

### 9.2 话术类型

#### **毒嘴型（sarcastic）**
- **风格**：犀利、激励
- **示例**："今天又重了？是不是又偷吃了？"
- **适用场景**：需要强烈激励的用户

#### **可爱型（cute）**
- **风格**：萌系、温暖
- **示例**："今天也坚持打卡了呢～好棒棒！"
- **适用场景**：喜欢温暖鼓励的用户

#### **鼓励型（encourage）**
- **风格**：正能量、支持
- **示例**："坚持就是胜利，你已经很棒了！"
- **适用场景**：需要持续鼓励的用户

#### **专业型（professional）**
- **风格**：数据、分析
- **示例**："今日体重变化-0.5kg，BMI下降0.2，继续保持"
- **适用场景**：关注数据的用户

#### **自定义（custom）**
- **风格**：用户自定义
- **功能**：用户可以自定义话术模板
- **适用场景**：有特殊需求的用户

## 十四、总结

### 14.1 核心价值

1. **打卡评价功能**：用户打卡后立即收到个性化评价反馈，提升打卡动力
2. **话术类型切换**：满足不同用户喜好，提升用户体验
3. **定时推送**：主动提醒用户，增强用户粘性

### 14.2 开发建议

1. **Phase 1**：优先实现绑定功能和打卡评价功能（核心价值）
2. **Phase 2**：实现话术类型切换和定时推送（重要功能）
3. **Phase 3**：实现机器人管理界面（增强功能）
4. **Phase 4**：实现自定义功能和数据分析（优化功能）

### 14.3 关键成功因素

- ✅ **简单易用**：绑定流程简单，一键复制即可完成
- ✅ **个性化**：支持多种话术类型，满足不同用户需求
- ✅ **及时响应**：打卡评价功能及时反馈用户
- ✅ **稳定可靠**：去重机制、错误处理确保稳定运行
